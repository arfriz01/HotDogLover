MVC Profile: Hot Dog Lover

March 03, 2015 by Alicia R. Frizzell

Welcome back! Today we will be talking about MVC and I will discuss a simple profile tutorial that you can try out to see how you like it. We will be using Visual Studio 2013 to make our MVC applications. Visual Studio is a great web development tool. It can be used to implement many different platforms such as ASP.NET, MVC and Web Forms.

The profile I have created this one for avid Hot Dog fans. I used a combination of models, views and controllers to display and control the data presented to the user.

"My Favorite Dog on the planet is the Cheddar Cheese Filled Dog at Tony's! You have to go try it out!"

We can allow the user to express their love for dogs with this application. They will be able to create their own profile where they can update information such as their profile picture, their favorite dog, or the place they ate. This application should have the functionality to create a hot dog and edit a user profile.

Models


The Model handles the logic for the application data.


This diagram shows the three pieces of this ASP.Net programming model.


To start our little tutorial, we need to create two models. These models will represent the application core. In this scenario we need to create two different models:

  • Hot Dog
  • Profile

In our Hot Dog model we can create the properties that each hotdog will possess. Below is an example of a Hot Dog class:

public class HotDog
    {
        public int HotDogID { get; set; }

        [Required(ErrorMessage = "Name is required")]
        [DisplayName("Hot Dog Name")]
        public string HotDogName { get; set; }
        [DisplayName("Last Ate")]
        [Required(ErrorMessage = "When you last ate it is required")]
        [DataType(DataType.Date)]
        [DisplayFormat(DataFormatString = "{0:MM/dd/yyyy}", ApplyFormatInEditMode = true)]
        public DateTime LastTimeAte { get; set; }

        [DisplayName("Where did you eat it?")]
        [Required(ErrorMessage = "Where you ate it is required")]
        public string LastPlaceAte { get; set; }
        [Range(1, 5,
            ErrorMessage = "Please Pick a rating")]
        public int Rating { get; set; }

    }

We can do the same thing for our Profile model:

namespace HotDogLover.Models
{
    public class Profile
    {
        public int ProfileID { get; set; }
        public string Name { get; set; }
        public string Bio { get; set; }
        public string Picture { get; set; }
        public HotDog FavoriteHotDog { get; set; }
        public List HotDogList { get; set; }
    }
}

Controller

The next concept we will cover is the Controllers.

The controller handles the input. For this portion of the application, we really only need to create one extra controller- a Profile Controller.

This part handles what the user inputs, so the profile controller will handle the input to the database records whenever a user tries to edit their information or add a new hot dog.

View

The View displays the data to the user in html form

Views make creating the layout of a site very simple. There is even a Shared folder if you wish to create something, such as a custom style sheet, that all views use.


For this application, I primarily used bootstrap to create the layout. I have not modified too much away from the original bootstrap template.
